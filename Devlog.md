<summary>26.05.06</summary>

# 작업 요약
- 새 프로젝트 시작 및 Core Library 분리 작업 착수.
  - 반복되는 초기 설정 시간을 단축하고, 프로젝트 간 코드 일관성을 유지하기 위한 기반 작업.
# 작업 상세
- 이전 프로젝트인 MiniRhythmStar에서 재사용 가능한 로직과 스크립트 선별.
- Core로 사용할 수 있게 리팩토링 및 구조 설계 작업 착수.
  - IBootable 인터페이스 도입을 통해 초기화 순서 제어 로직 규격화.
  - 시스템 초기화 시 구체적인 객체 타입을 직접 캐싱할 필요가 없어지면서 모듈 간 의존성을 낮춤.
- 완성 이후 Unity Package Manager로 구성해 별개의 레포지토리로 만들 예정.
- 프로젝트 디렉토리 구조 표준화 : 기능과 역할에 따른 코드 배치를 위해 기초 폴더 구조 설계 완료.
# 주석
새 프로젝트를 시작하기에 앞서, 앞으로의 프로젝트에서도 시작할 때 사용할 뼈대부터<br>
체계적으로 관리할 수 있으면 좋겠다는 생각이 들었기에 이번엔 시작부터 구조를 제대로 잡고 시작해보려 한다.<br>
또, 이전 프로젝트에서도 폴더를 나누려는 노력은 했으나,<br>
규모가 커지면서 원하는 코드를 한 번에 찾기 어려워지는 한계가 있었다.<br>
이번에는 시작부터 디렉토리 구조를 잡고 가는 것으로 유지보수 효율을 올려 보려고 한다.

# 작업 요약
- 코어 시스템 완성 및 패키지화하여 별도 레포지토리 분리
  - 새 프로젝트에서 리팩토링된 코어 시스템이 잘 작동하는지 확인하는 것이 새로운 목표가 될 것으로 보임.
# 작업 상세
- 코어 라이브러리와 재사용 가능한 Utils 분리 작업 완료.
- 새 프로젝트에 적용해보고, 한 차례 더 리팩토링 작업.
  - 새 프로젝트에서 SoundData와 SceneNames를 세팅하기 쉽도록, 프로젝트별 데이터와 Core 시스템 분리.
  - 프로젝트별 Scene/Sound enum을 Core와 분리하고 인스펙터 드롭다운 기반 설정 구조 완성.
- GitHub UPM 패키지 형태로 Core 시스템을 분리했고, 다음 프로젝트에서도 Git URL로 가져와 사용할 수 있게 만듬.
# 주석
프로젝트의 초기 주요 목적은 애니메이션 기능 학습이었으나<br>
새 프로젝트를 시작하는 데 사용할 공통된 파이프라인을 만들기 위해<br>
기존의 쓰던 코드들을 대거 리팩토링하게 되었고,<br>
이번 프로젝트에서 코어 시스템이 문제 없이 작동할 수 있게 하는 것도 주요 과제가 될 것 같다.
