## 5. 주요 기능
### 5.1 마우스 방향 기반 차지 점프
유저의 키 입력 시간에 비례해 점프력을 증가시키고, 마우스 방향에 따라 점프 방향을 결정한다.
- 차지 시간에 따른 점프 힘 증가
- 최대 차지 시간 제한
- 마우스 위치 기반 점프 방향 계산
- 점프 가능 상태와 공중 상태 분리
- 점프 시 애니메이션, 사운드, 이펙트 연동
### 5.2 플랫폼 자동 생성 및 풀링
유저가 유효한 플랫폼에 착지하는 것을 감지하고 오브젝트 풀링을 활용해 플랫폼을 생성, 회수할 수 있다.
- 프리팹 배리언트를 활용해 크기별로 프리팹 그룹 관리
- 플랫폼 오브젝트 풀링
- 오래된 플랫폼 자동 회수
- 플랫폼 건너뜀 케이스 방어
### 5.3 점수, 콤보, 보너스 시스템
유저 플랫폼 착지 시 정보를 기반으로 점수와 보너스를 증감, UI 팝업과 갱신 등을 처리한다.
- 첫 방문 플랫폼 착지 시 점수 증가
- 이미 방문한 플랫폼 또는 뒤로 간 플랫폼 착지 시 콤보/보너스 초기화
- 플랫폼 크기별 점수 규칙 테이블
- 점수 획득, 보너스 증감 팝업 출력
### 5.4 카메라 포커스 및 줌
유저의 위치를 포커스하는 기능과 마우스 휠에 따라 줌인, 줌아웃 기능, 추후 코어 패키지로의 확장을 위해 코어 기능과 게임 로직을 분리했다.
- 플레이어 위치 기반 카메라 포커스
- 마우스 휠 기반 Orthographic Size 조절
- 코어 로직과 게임 로직 분리
### 5.5 기타
그 외에 유저 위치를 기반으로 한 배경 타일링 로직과 유저의 리지드바디의 중력, 속도를 제어하는 기능을 포함한 일시정지 기능, 게임 재시작 기능.
- 배경 타일링
- 일시정지 기능
- 재시작 기능
## 6. 아키텍처

## 8. 고민했던 부분들
샘플 프로젝트에서는 점프 방향이 고정되어 있었고, 유저는 앞으로만 나아갈 수 있었기에 고려되지 않은 부분들이 많았다.
- 유저가 역주행을 할 경우 배경과 플랫폼은 어떻게 할 것인가?
- 같은 곳에 착지하거나, 플랫폼을 건너뛸 경우의 처리는 어떻게 할 것인가?

이를 해결하기 위해 각 플랫폼에 방문 여부와 인덱스 번호를 부여해 이를 통해 inValid 조건을 판별했고<br>
배경 타일링 로직을 구현, 오브젝트 풀링에서는 조건을 플랫폼 스폰 세팅에서 한 번에 조절할 수 있도록 ScriptableOBj로 만들었다.
